Also renamed containers to match `ruby-core` convention: `{product}-{env}-{service}` → `navi-prod-digest`, `navi-staging-digest`, `navi-dev-digest`. → Cut `v0.1.4`; **full pipeline green** for the first time.

## Open questions

1. **SMS delivery:** `notify-sms.sh` exits 0 silently when Vault is unreachable (Twilio not seeded). No SMS confirmation of deploy success/failure until Twilio is configured in P1.

2. **Node.js 20 deprecation:** All GitHub Actions steps warn about Node.js 20 deprecation (forced to Node.js 24 in June 2026). `actions/checkout@v4`, `docker/login-action@v3` have updates available. Not blocking but must be addressed before June 2026.

3. **`.last-deployed-version` dual-purpose tension:** The file serves as both rollback target and change detection baseline. When a deploy fails and rolls back, the file stays at the last successful version — which can cause the change detection to see a net-zero diff if subsequent changes cancel out. Observed with v0.1.5→v0.1.8. Should be tracked as a P1 architecture note.
